## 1. 소프트웨어 분류

### IMDRF SaMD 분류

| 항목 | 분류 |
|------|------|
| **SaMD 여부** | 해당 (Software as a Medical Device) |
| **IMDRF 위험 범주** | Category I (낮은 위험) |
| **IEC 62304 소프트웨어 안전 등급** | Class B |
| **의도된 용도** | 4D Flow MRI 의료 영상 분석 보조 — 영상 업로드, 분석 업무 관리, 리포트 제공 |
| **최종 의사결정권** | 의료진 (소프트웨어는 의사결정 보조 도구) |

**Class B 근거**:
- 소프트웨어 고장이 환자에게 직접적인 위해를 야기하지 않음
- 의료 영상 *분석 보조* 도구이며 진단·치료 결정은 의료진이 수행
- 소프트웨어 오류 시 의료진이 육안으로 재검토 가능

## 3. IEC 62304 준수 현황

### 3.1 소프트웨어 개발 계획

| 활동 | 구현 방식 |
|------|---------|
| 요구사항 관리 | GitHub Issues / 내부 문서 (`docs/10_CERTIFICATION.md`) |
| 아키텍처 설계 | `docs/01_ARCHITECTURE.md` |
| 세부 설계 | `docs/08_FEATURES.md`, API 문서 (`docs/03_API.md`) |
| 구현 | TypeScript + ESLint로 코드 품질 관리 |
| 단위 테스트 | Jest (`npm test`) |
| 통합 테스트 | 수동 QA 체크리스트 |
| 소프트웨어 릴리스 | git 태그 기반 버전 관리, `CHANGELOG.md` |

### 3.2 소프트웨어 아이템 식별

| 소프트웨어 아이템 | 설명 | 위험 등급 |
|--------------|------|---------|
| 인증 모듈 | JWT 발급·검증, bcrypt 해싱 | 중간 (무단 접근 시 PHI 노출) |
| RBAC 모듈 | 역할 기반 접근 제어 | 중간 (권한 오류 시 데이터 노출) |
| NIfTI 파싱 모듈 | 의료 영상 파일 파싱·렌더링 | 낮음 (분석 보조, 의료진이 재검토) |
| 파일 관리 모듈 | S3 업로드·다운로드·삭제 | 낮음 (파일 손실 시 재업로드 가능) |
| 업무 관리 모듈 | 업무 배정·상태 관리 | 낮음 (업무 지연이 환자 직접 위해 아님) |
| 감사 로그 모듈 | 액션 이력 기록 | 낮음 |

### 3.3 형상 관리

- **VCS**: Git (GitHub)
- **브랜치 전략**: `main` (production), `develop` (개발), feature 브랜치
- **릴리스 태그**: `v{major}.{minor}.{patch}` 형식 (Semantic Versioning)
- **변경 이력**: `CHANGELOG.md`

## 4. ISO 14971 위험 관리

### 4.1 위험 식별 및 평가

| 위험 | 위험 원인 | 위험 결과 | 위험 수준 | 위험 저감 조치 |
|------|---------|---------|---------|--------------|
| **무단 접근** | JWT 탈취 또는 인증 우회 | PHI 노출, 의료 데이터 변조 | 중간 | HttpOnly 쿠키, Rate Limiting, RBAC |
| **데이터 손실** | DB 장애 또는 S3 삭제 | 케이스·리포트·영상 파일 손실 | 중간 | Aurora 자동 스냅샷(7일), S3 11 9s 내구성 |
| **분석 오류** | NIfTI 파싱 버그 | 잘못된 영상 렌더링 | 낮음 | 오픈소스 표준 라이브러리 사용, 의료진 재검토 |
| **SQL Injection** | 파라미터 미검증 | DB 전체 노출·조작 | 높음 | Prepared Statement 전용 강제 |
| **XSS** | HTML 미sanitize | 브라우저 내 악성 스크립트 실행 | 중간 | DOMPurify, HttpOnly 쿠키 |
| **서버 다운** | OOM 또는 프로세스 충돌 | 서비스 중단 | 낮음 | PM2 자동 재시작, `max_memory_restart` |
| **파일 만료 오류** | 첨부파일 7일 정책 미적용 | 만료된 파일 접근 불가 | 낮음 | `task_file_attachments.uploaded_at` 기준 관리 |

### 4.2 잔여 위험 수용 기준

| 항목 | 기준 |
|------|------|
| 소프트웨어는 의료 의사결정 보조 도구이며 최종 결정은 의료진이 수행 | ✅ 수용 |
| 잔여 위험이 임상적 이익보다 낮음 | ✅ 수용 |
| 표준 보안 조치 적용 후 잔여 위험은 합리적 수준 | ✅ 수용 |

## 5. 보안·데이터 보호

### 5.1 PHI 처리 방침

| 항목 | 구현 |
|------|------|
| 환자 정보 저장 위치 | Aurora MySQL (`cases.patient_name`, `cases.study_date`만 저장) |
| 파일 스토리지 | AWS S3 버킷 (ap-northeast-2) — 퍼블릭 액세스 전체 차단 |
| 전송 암호화 | HTTPS (TLS 1.2/1.3) 필수 |
| 저장 암호화 | Aurora 저장 시 암호화, S3 SSE 활성화 |
| 접근 제어 | RBAC — client는 본인 케이스만 접근 가능 |
| 감사 추적 | `audit_log` 테이블에 모든 주요 액션 기록 |

### 5.2 데이터 보존 정책

| 데이터 유형 | 보존 기간 | 근거 |
|-----------|---------|------|
| 케이스·리포트 | 무기한 (수동 삭제) | 의료 기록 규제 준수 |
| 업무 첨부파일 | 7일 (`task_file_attachments.uploaded_at` 기준) | 임시 파일 |
| 감사 로그 | 무기한 | 규제 준수·책임 추적 |
| Staff 가입 대기 | 처리 완료 후 삭제 | 개인정보 최소 보존 |

## 6. 추적성 매트릭스

| 요구사항 | 구현 위치 | 테스트 위치 |
|---------|---------|----------|
| 사용자 인증 | `lib/database/auth.ts`, `lib/auth/index.ts` | `app/api/auth/*/route.ts` |
| 역할 기반 접근 제어 | `lib/auth/index.ts`, 각 API Route | 수동 QA |
| PHI 보호 | Next.js 서버 컴포넌트, `cases` 테이블 접근 제어 | 수동 QA |
| 파일 안전 저장 | `lib/services/aws/s3.ts`, S3 버킷 정책 | AWS 콘솔 확인 |
| 감사 추적 | `lib/db/audit.ts`, `audit_log` 테이블 | DB 쿼리 확인 |
| SQL Injection 방지 | `lib/database/mysql.ts query()` | 코드 리뷰 |
| XSS 방어 | `components/safe-html.tsx`, HttpOnly 쿠키 | 수동 QA |
| 비밀번호 보안 | bcryptjs cost 10, Secrets Manager 로테이션 | 코드 리뷰 |
